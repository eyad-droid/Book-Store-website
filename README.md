# Book-Store-website
Welcome to our **Book Store Website** project! This repository is dedicated to our **Information Technology (IT)** course project. Our team consists of **8 members**, each handling a specific part of the development process to ensure a high-quality, functional, and well-structured website.
## 🛠️ Project Structure & Roadmap
The project is divided into multiple interconnected modules, focusing on clean code, semantic HTML, and advanced CSS/JS features as per the course requirements.
### 👥 Our Team & Responsibilities
| Role | Responsibility & Tasks |
|---|---|
| **Team Leader / Integrator** | Managing project structure, page linking, code reviews, GitHub management (branches & merging), and overall project delivery. |
| **UI Designer (CSS Master)** | Responsible for the visual identity: colors, spacing, fonts, and global CSS styling across all 8 pages. |
| **Theme System (Light/Dark)** | Implementing the theme toggle switch, saving preferences in localStorage, and ensuring the theme persists on page reload. |
| **Forms & Validation** | Developing Login, Register, and Contact forms with JavaScript validation for a smooth user experience. |
| **Storage & Data Handler** | Managing user data and preferences using localStorage (Reading/Writing data). |
| **Pages Developer (1)** | Building the core pages: **Home, About, and Services** using semantic HTML. |
| **Pages Developer (2)** | Developing content-rich pages: **Products/Gallery and Blog**, incorporating Image Maps and iframes. |
| **Media & Advanced Elements** | Integrating advanced Tables, Video/Audio elements, and Iconography. |
## 📝 Guidelines & Rules
To maintain the quality and integrity of our work, the team has agreed on the following:
 * **Documentation:** Clear **Comments** must be written throughout the code to help fellow teammates understand the logic.
 * **Knowledge Sharing:** After completion, we will hold an internal meeting where every member explains their part to the rest of the team.
 * **Collaboration:** Even though tasks are assigned individually, we work as one unit. Support is always available for anyone facing technical challenges.
